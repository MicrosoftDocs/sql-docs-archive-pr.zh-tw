 <span data-ttu-id="f10b9-109">若要從 SSIS 套件叫用原生編譯的預存程序，則建議使用 ODBC 來源或 ODBC 目的地，其包含的 SQL 陳述式具有下列格式：不含 **EXEC** 關鍵字的 **\<procedure name>** 。</span><span class="sxs-lookup"><span data-stu-id="f10b9-109">To invoke a natively compiled stored procedure from an SSIS package, we recommend that you use an ODBC Source or ODBC Destination with an SQL statement of the format: **\<procedure name>** without the **EXEC** keyword.</span></span> <span data-ttu-id="f10b9-110">如果您在 SQL 陳述式中使用 EXEC 關鍵字，則會看見錯誤訊息，因為 ODBC 連接管理員會將 SQL 命令文字解譯為 [!INCLUDE[tsql](../../includes/tsql-md.md)] 陳述式而非預存程序，且將使用執行原生編譯的預存程序時不支援的資料指標。</span><span class="sxs-lookup"><span data-stu-id="f10b9-110">If you use the EXEC keyword in the SQL statement, you will see an error message because the ODBC connection manager interprets the SQL command text as a [!INCLUDE[tsql](../../includes/tsql-md.md)] statement rather than a stored procedure and use cursors, which are not supported for execution of natively compiled stored procedures.</span></span> <span data-ttu-id="f10b9-111">連接管理員會將不含 EXEC 關鍵字的 SQL 陳述式視為預存程序呼叫，而且不會使用資料指標。</span><span class="sxs-lookup"><span data-stu-id="f10b9-111">The connection manager treats the SQL statement without the EXEC keyword as a stored procedure call and will not use a cursor.</span></span>  
  
 <span data-ttu-id="f10b9-112">您也可以使用 ADO .NET 來源和 OLE DB 來源叫用原生編譯預存程序，不過我們建議您使用 ODBC 來源。</span><span class="sxs-lookup"><span data-stu-id="f10b9-112">You can also use ADO .NET Source and OLE DB Source to invoke a natively compiled stored procedure, but we recommend that you use ODBC Source.</span></span> <span data-ttu-id="f10b9-113">如果您設定由 ADO .NET 來源執行原生編譯的預存程序，則會看見錯誤訊息，因為 [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] (SqlClient) 的資料提供者 (其為 ADO .NET 來源預設使用) 不支援執行原生編譯的預存程序。</span><span class="sxs-lookup"><span data-stu-id="f10b9-113">If you configure the ADO .NET Source to execute a natively compiled stored procedure, you will see an error message because the data provider for [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] (SqlClient), which the ADO .NET Source uses by default, does not support execution of natively compiled stored procedures.</span></span> <span data-ttu-id="f10b9-114">您可以將 ADO .NET 來源設定為使用 ODBC 資料提供者、OLE DB Provider for [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)]或 [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Native Client。</span><span class="sxs-lookup"><span data-stu-id="f10b9-114">You can configure the ADO .NET Source to use the ODBC Data Provider, OLE DB Provider for [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)], or [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Native Client.</span></span> <span data-ttu-id="f10b9-115">不過，請注意，ODBC 來源的執行效能比使用 ODBC 資料提供者的 ADO .NET 來源更佳。</span><span class="sxs-lookup"><span data-stu-id="f10b9-115">However, note that the ODBC Source performs better than ADO .NET Source with ODBC Data Provider.</span></span>
