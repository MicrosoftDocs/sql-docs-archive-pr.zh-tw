# <a name="manage-checkins"></a>管理簽入
  若要變更其他使用者能夠使用的原始檔控制檔案，您必須簽入檔案。 當您簽入檔案時，您建立的版本會複製到原始檔控制提供者中，且會成為檔案的最新版本，有適當權限的使用者通常都可以使用它。  
  
 用 [!INCLUDE[msCoName](../includes/msconame-md.md)] [!INCLUDE[ssManStudioFull](../includes/ssmanstudiofull-md.md)] 來簽入檔案。 當您需要定期更新原始檔控制存放區，但又需要維護對某一組檔案的控制時，您可以指定您簽入的檔案，對您而言，要維持簽出的狀態。  
  
 下表描述此章節的主題。  
  
|主題|描述|  
|-----------|-----------------|  
|[簽入檔案](../../2014/database-engine/check-in-files.md)|提供如何簽入您已修改之檔案的指示。|  
|[檢視修改檔案清單](../../2014/database-engine/view-a-list-of-modified-files.md)|說明如何顯示工作完成時能夠一起簽入的修改檔案清單。|  
|[編輯簽入的檔案](../../2014/database-engine/edit-checked-in-files.md)|說明如何設定 [!INCLUDE[ssManStudioFull](../includes/ssmanstudiofull-md.md)] 環境，以便修改尚未簽出的檔案。|
